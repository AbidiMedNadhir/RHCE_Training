# 0. Introduction to RHCE and Ansible Automation

## Introduction
Bienvenue dans ce cours de préparation à la certification **Red Hat Certified Engineer (RHCE – EX294)**.

---

## Qu’est-ce qu’Ansible ?
Ansible est un outil d’automatisation **open-source** utilisé pour :
- la gestion de configuration,
- le déploiement d’applications,
- l’exécution de tâches administratives répétitives.

Ansible est dit **agentless**, ce qui signifie qu’aucun logiciel spécifique n’est requis sur les machines gérées.  
Il communique principalement via **SSH**, ce qui le rend simple, sécurisé et rapide à déployer.

Ansible utilise des fichiers **YAML**, faciles à lire et à écrire, ce qui permet aux administrateurs système de créer des automatisations claires et maintenables.

## Architecture d’Ansible

L’architecture d’Ansible est simple et repose sur les composants suivants :

### Control Node (Ansible Control Machine)
Le control node est la machine où Ansible est installé.  
C’est depuis cette machine que vous :
- écrivez les playbooks,
- gérez l’inventaire,
- exécutez les commandes Ansible.

Dans l’examen RHCE, le control node est fourni et Ansible y est déjà installé.

---

### Managed Nodes (Target Nodes)
Les managed nodes sont les serveurs cibles que vous gérez avec Ansible.

Caractéristiques importantes :
- Ansible n’a pas besoin d’être installé sur ces machines
- Python et SSH doivent être disponibles
- Les commandes sont exécutées à distance depuis le control node

---

### Inventory
L’inventaire est un fichier qui contient la liste des machines gérées par Ansible.

Il permet :
- de définir des groupes de serveurs,
- d’organiser l’infrastructure,
- de cibler des hôtes spécifiques dans les playbooks.

L’inventaire peut être statique (fichier texte) ou dynamique.

---

### Playbooks
Les playbooks sont des fichiers YAML qui décrivent **l’état souhaité** des systèmes.

Un playbook contient :
- des hôtes ciblés,
- des tâches,
- des modules,
- des variables.

Les playbooks sont **le cœur de l’examen RHCE**.

---

### Modules
Les modules sont les unités de travail d’Ansible.

Chaque tâche appelle un module pour effectuer une action précise, par exemple :
- installer un paquet,
- gérer un service,
- copier un fichier,
- créer un utilisateur.

Ansible fournit des centaines de modules prêts à l’emploi.

---

### Tasks, Roles et Handlers
- **Tasks** : actions individuelles exécutées sur les hôtes
- **Roles** : structure standard pour organiser et réutiliser les playbooks
- **Handlers** : tâches spéciales déclenchées uniquement lorsqu’un changement se produit (ex: redémarrer un service)

Les rôles sont fortement utilisés dans l’examen RHCE.

---

## Comment fonctionne Ansible ?
Le fonctionnement d’Ansible suit les étapes suivantes :

1. L’administrateur écrit un inventaire et des playbooks sur le control node
2. Une connexion SSH sans mot de passe est établie vers les managed nodes
3. Ansible collecte des informations sur les systèmes (facts)
4. Ansible envoie les instructions aux managed nodes
5. Les tâches sont exécutées et les résultats sont retournés au control node

## Environnement de travail pour ce cours

Pour suivre ce cours dans des conditions proches de l’examen RHCE, l’environnement recommandé est :

- 3 machines Linux :
  - 1 Control Node
  - 2 Target Nodes
- Systèmes :
  - Red Hat Enterprise Linux 9 ou CentOS Stream
- Python 3 installé sur toutes les machines
- Connexion SSH fonctionnelle

Cet environnement permet de pratiquer tous les scénarios exigés dans l’examen.
